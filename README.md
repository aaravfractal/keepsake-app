## The problem

Your most personal moments live inside systems you don't own. ChatGPT memory, companion apps, journaling tools — they can delete your memories, mine them for ads, or shut down and take everything with them. You pour your life into a black box you don't control.

## What Keepsake does

You talk to Keepsake and share moments, events, and feelings. It stores each one as an encrypted, user-owned asset, then recalls and reflects on them — writing your memories back to you as stories and insights. Every memory is provably, permanently yours.

The loop:
1. **Capture** — share a memory in a calm chat interface.
2. **Encrypt + store** — encrypted on-device, written to 0G Storage as a user-owned asset, recorded onchain.
3. **Recall + reflect** — ask a question; Keepsake decrypts your relevant memories and runs TEE inference to write back a reflective story.
4. **Prove** — every reflection carries a provenance certificate: which memories it drew from, a sealed-inference receipt, all verifiable onchain.

## 0G integration (load-bearing across all three layers)

Keepsake runs on 0G — it isn't bolted on:

- **0G Storage** — every memory is encrypted client-side (ECIES, encrypt-to-self) and stored as a user-owned asset. 0G holds only ciphertext; only your key decrypts it. Each memory returns an onchain root hash.
- **0G Compute (TEE)** — reflections are generated by TEE inference, not a database read. Each carries an on-chain provider trace and request ID — a verifiable "sealed inference" receipt.
- **0G Chain** — provenance for every memory (root hash, owner, timestamp), and the proof tying each reflection to the specific onchain memories it used.

Remove any one and Keepsake stops being itself: no Storage means no ownership, no Compute means no reflection, no Chain means no proof.

## Verifiable by design

The provenance certificate makes ownership tangible — open it on any reflection to see the source memories' root hashes, the TEE compute provider, and the request ID. You can always prove a memory is yours and unaltered, and that a reflection came from your real memories rather than a model's imagination.

## Security

- Client-side ECIES encryption before anything leaves the device; 0G holds only ciphertext.
- Encrypt-to-self with the owner's key — the operator cannot read memories.
- No third-party credentials held; no secrets in the repo.
